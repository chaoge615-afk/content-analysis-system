# 智能内容分析系统 - 前端功能说明

> 版本：1.0.0
> 更新时间：2026-05-25
> 访问地址：http://localhost（Docker 部署）或 http://localhost:3000（开发模式）

前端采用 **顶部 Tab 切换** 的双视图架构，无需页面跳转：

```

├──────────────────────────────────────────────────────────┤

│   对话视图                    管理面板视图                │
│   ┌────────────┬────────┐    ┌──────────────────────┐   │
│   │            │  侧    │    │  [采集触发] [查询日志] │   │

│                                                          │
```

### 技术栈
- **框架**：React 18 + TypeScript
- **样式**：Tailwind CSS 3.4
- **构建**：Vite 5
- **HTTP**：Axios（2 分钟超时）
- **部署**：Nginx 反向代理（`:80` → 前端 `:3000` + API `:8000`）

## 二、对话视图

### 2.1 智能问答

主聊天区域，支持自然语言提问，系统自动判断查询路由：

| 路由类型 | 标签颜色 | 适用场景 | 示例问题 |
|---------|---------|---------|---------|
| **结构化** (structured) | 蓝色 | 数据统计、计数、排名 | "桃姐最近发了几个视频？" |
| **语义** (semantic) | 绿色 | 观点、建议、内容检索 | "博主们对冷暴力怎么看？" |
| **混合** (hybrid) | 紫色 | 同时需要数据+内容 | "桃姐最近聊了什么？她的建议？" |

每条 AI 回复展示：
- **回答内容**：Markdown 格式渲染
- **路由标签**：显示查询走的通道（结构化/语义/混合）
- **推理过程**：可折叠的意图分析说明
- **SQL 语句**：结构化/混合模式下可展开查看生成的 SQL
- **SQL 结果**：表格形式展示原始查询数据
- **响应时间**：右下角显示耗时（秒）

### 2.2 斜杠命令

在输入框中以 `/` 开头触发快捷命令：

| 命令 | 功能 | 示例 |
|------|------|------|
| `/status` | 查看系统状态（Router / SQL / RAG） | `/status` |
| `/up_list` | 查看所有 UP 主列表及视频数 | `/up_list` |
| `/recent` | 查看最近采集的 10 个视频 | `/recent` |
| `/categories` | 查看 31 个情感分类统计 | `/categories` |
| `/sql [问题]` | 强制走 Text-to-SQL 通道 | `/sql 各分类视频数量` |
| `/rag [问题]` | 强制走 RAG 通道 | `/rag 关于分手的建议` |
| `/clear` | 清空当前对话 | `/clear` |
| `/help` | 显示命令帮助 | `/help` |

### 2.3 右侧边栏

可折叠的信息面板（通过右上角"隐藏面板/显示面板"按钮切换）：

- **数据概览**：视频总数、UP 主数量、知识库文档数、分类数
- **快捷问题**：预设的示例问题按钮，点击直接发送查询
- **系统状态**：各子服务运行状态指示

## 三、管理面板

顶部 Tab 切换到"管理面板"后，展示三个子功能页签。

### 3.1 采集触发

手动触发 bilibili-monitor 采集任务，无需登录服务器。

#### 功能说明

| 功能 | 说明 |
|------|------|
| **参数配置** | 可选填 `最大视频数`（限制采集量）和 `UP主名称`（指定采集对象） |
| **开始采集** | 点击按钮后通过 Docker SDK 启动 bilibili-monitor 容器 |
| **状态指示** | 实时显示任务状态：空闲(灰) / 运行中(蓝,脉冲) / 完成(绿) / 失败(红) |
| **实时日志** | 终端风格的日志滚动窗口（深色背景、绿色字体），5 秒自动刷新 |
| **互斥控制** | 同时只能运行一个采集任务，运行中时按钮禁用 |

#### 日志显示

```

```

### 3.2 查询日志

展示所有通过 `/api/chat` 发起的查询记录，支持分页和过滤。

#### 统计卡片（顶部）

| 卡片 | 内容 |
|------|------|
| **总查询数** | 历史累计查询次数 |
| **平均响应时间** | 所有查询的平均耗时（秒） |
| **结构化 / 语义 / 混合** | 各路由类型的查询次数和占比 |

#### 查询记录表格

| 列名 | 说明 |
|------|------|
| **问题** | 用户提问（截断显示，点击行可展开完整内容） |
| **路由类型** | 彩色标签（蓝=结构化 / 绿=语义 / 紫=混合） |
| **响应时间** | 本次查询耗时（秒） |
| **查询时间** | 发起查询的时间戳 |

#### 交互功能
- **过滤**：顶部按钮组按路由类型过滤（全部 / 结构化 / 语义 / 混合）
- **分页**：底部翻页控件，每页 20 条
- **展开**：点击行展开查看完整问题文本

### 3.3 服务监控

实时展示各 Docker 容器的运行状态和资源使用情况。

#### 容器状态卡片

每个服务一张卡片，展示：

| 指标 | 说明 |
|------|------|
| **状态灯** | 绿色圆点 = running / 红色 = stopped / 灰色 = 未知 |
| **容器名称** | Docker 容器名（如 `ai项目-chromadb-1`） |
| **内存使用** | 进度条 + 数值（如 `512 MB / 1 GB`） |
| **CPU 使用率** | 百分比显示 |

监控的 5 个服务：
1. **chromadb**（端口 8001）— 向量数据库
2. **text-to-sql**（端口 8010）— 结构化查询引擎
3. **rag**（端口 8090）— 语义检索引擎
4. **router-agent**（端口 8000）— 路由 Agent
5. **frontend**（端口 80）— 前端 + Nginx

#### 知识库指标

| 指标 | 数据来源 |
|------|---------|
| **向量文档块数** | ChromaDB `video_knowledge` collection |
| **视频总数** | DuckDB `video_meta` 表 |
| **总查询数** | DuckDB `query_log` 表 |

#### 查询类型分布

纯 CSS 实现的水平条形图，展示结构化 / 语义 / 混合三种路由的查询占比，带颜色编码和百分比标注。

#### 刷新控制
- **自动刷新**：默认 30 秒自动刷新，可开关
- **手动刷新**：点击刷新按钮立即更新数据

## 四、API 接口

前端调用的所有后端接口（通过 Nginx 反向代理转发到 router-agent）：

### 核心接口

| 方法 | 路径 | 功能 |
|------|------|------|
| `POST` | `/api/chat` | 统一问答入口（body: `{ question, force_route? }`) |
| `GET` | `/api/status` | 系统状态（各子系统可用性） |
| `GET` | `/api/up_list` | UP 主列表 |
| `GET` | `/api/recent` | 最近采集视频 |
| `GET` | `/api/categories` | 分类统计 |

### 管理接口

| 方法 | 路径 | 功能 |
|------|------|------|
| `POST` | `/api/trigger_monitor` | 触发采集任务（body: `{ max_videos?, up_name? }`) |
| `GET` | `/api/trigger_status` | 采集任务状态 + 实时日志 |
| `GET` | `/api/query_stats` | 查询统计（query: `page, page_size, route_type`） |
| `GET` | `/api/system_metrics` | 系统指标（容器资源 + 知识库 + 查询统计） |

## 七、开发信息

| 项目 | 说明 |
|------|------|
| 源码位置 | `text-to-sql/frontend/` |
