# 智能内容分析系统 - 前端功能说明

> 版本：2.0.0
> 更新时间：2026-05-28
> 访问地址：http://localhost（Docker 部署）或 http://localhost:3000（开发模式）

---

## 一、整体布局

前端采用 **顶部 Tab 切换** 的双视图架构，无需页面跳转：

```
┌──────────────────────────────────────────────────────────┐
│  智能内容分析系统        [ 对话 ]  [ 管理面板 ]            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   对话视图                    管理面板视图                │
│   ┌────────────┬────────┐    ┌──────────────────────┐   │
│   │            │  侧    │    │ [采集] [GPU] [日志]   │   │
│   │   聊天区域  │  边    │    │ [监控]               │   │
│   │            │  栏    │    │                      │   │
│   ├────────────┤        │    │    功能内容区         │   │
│   │  输入框     │        │    │                      │   │
│   └────────────┴────────┘    └──────────────────────┘   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

管理面板包含 **4 个子功能页签**：📡 采集触发、🎙️ GPU转录、📋 查询日志、📊 服务监控。

### 技术栈
- **框架**：React 18 + TypeScript
- **样式**：Tailwind CSS 3.4
- **构建**：Vite 5
- **HTTP**：Axios（2 分钟超时，GPU 服务 5 分钟超时）
- **部署**：Nginx 反向代理（`:80` → 前端静态文件 + API `:8000`）

### 组件清单

| 组件 | 文件 | 说明 |
|------|------|------|
| App | `App.tsx` | 应用外壳，Tab 切换 + 侧边栏 |
| ChatInput | `ChatInput.tsx` | 输入框，支持斜杠命令 |
| ChatMessage | `ChatMessage.tsx` | 聊天消息气泡，路由标签+SQL+推理 |
| QuickView | `QuickView.tsx` | 斜杠命令结果结构化展示 |
| StatusPanel | `StatusPanel.tsx` | 右侧边栏（数据概览+快捷问题+状态） |
| AdminPanel | `AdminPanel.tsx` | 管理面板容器，4 个子 Tab |
| MonitorTrigger | `MonitorTrigger.tsx` | 采集触发 + Cookie 管理 + UP主管理 |
| UpManager | `UpManager.tsx` | UP主添加/删除/列表 |
| GpuTranscribe | `GpuTranscribe.tsx` | GPU 转录 + ASR 云转写 |
| QueryLog | `QueryLog.tsx` | 查询日志分页列表 |
| SystemMetrics | `SystemMetrics.tsx` | 服务监控仪表盘 |
| ResultTable | `ResultTable.tsx` | SQL 结果表格渲染 |

## 二、对话视图

### 2.1 智能问答

主聊天区域，支持自然语言提问，系统自动判断查询路由：

| 路由类型 | 标签颜色 | 适用场景 | 示例问题 |
|---------|---------|---------|---------|
| **结构化** (structured) | 蓝色 | 数据统计、计数、排名 | "桃姐最近发了几个视频？" |
| **语义** (semantic) | 绿色 | 观点、建议、内容检索 | "博主们对冷暴力怎么看？" |
| **混合** (hybrid) | 紫色 | 同时需要数据+内容 | "桃姐最近聊了什么？她的建议？" |

每条 AI 回复展示：
- **回答内容**：Markdown 格式渲染
- **路由标签**：显示查询走的通道（结构化/语义/混合）
- **推理过程**：可折叠的意图分析说明
- **SQL 语句**：结构化/混合模式下可展开查看生成的 SQL
- **SQL 结果**：表格形式展示原始查询数据
- **响应时间**：右下角显示耗时（秒）
- **错误提示**：当 LLM 额度不足或网络异常时，回答前显示 ⚠️ 警告提示

### 2.2 斜杠命令

在输入框中以 `/` 开头触发快捷命令：

| 命令 | 功能 | 示例 |
|------|------|------|
| `/status` | 系统状态（Router / SQL / RAG） | `/status` |
| `/up_list` | 所有 UP 主列表及视频数 | `/up_list` |
| `/recent` | 最近采集的 10 个视频 | `/recent` |
| `/categories` | 31 个情感分类统计 | `/categories` |
| `/sql [问题]` | 强制走 Text-to-SQL 通道 | `/sql 各分类视频数量` |
| `/rag [问题]` | 强制走 RAG 通道 | `/rag 关于分手的建议` |
| `/clear` | 清空当前对话 | `/clear` |
| `/help` | 显示命令帮助 | `/help` |

斜杠命令结果通过 `QuickView` 组件结构化展示：
- `/status`：三色状态卡片（Router / SQL / RAG 可用性）
- `/up_list`：UP主卡片列表（头像 + 名称 + 视频数）
- `/recent`：视频列表（标题 + UP主 + 分类 + 时长标签）
- `/categories`：水平条形图（分类名 + 视频数 + 百分比）

### 2.3 右侧边栏

可折叠的信息面板（通过右上角按钮切换）：

- **数据概览**：视频总数、UP 主数量、知识库文档数、分类数
- **快捷问题**：预设的示例问题按钮，点击直接发送查询
- **系统状态**：各子服务运行状态指示

## 三、管理面板

顶部 Tab 切换到"管理面板"后，展示四个子功能页签。

### 3.1 采集触发

手动触发 bilibili-monitor 采集任务，管理 Cookie 和 UP主配置。

#### 状态指示器

页面顶部显示采集状态：
- **空闲**：灰色圆点
- **运行中**：蓝色脉冲动画 + 开始时间
- **完成**：绿色圆点 + 容器名 + 退出码
- **失败**：红色圆点 + 错误信息

#### Cookie 管理

可折叠的 Cookie 设置面板，状态灯指示配置状态：

| 状态 | 显示 | 说明 |
|------|------|------|
| 已配置 | 🟢 绿色 + "已配置"标签 | Cookie 文件存在且包含有效字段 |
| 已过期 | 🔴 红色 + "已过期"标签 | 测试发现 Cookie 已失效 |
| 未配置 | 🔴 红色 + "未配置"标签 | 无 Cookie 文件 |

操作按钮：
- **保存 Cookie**：粘贴 Netscape 格式的 Cookie 内容，保存到持久化文件
- **测试 Cookie**：调用 B站 nav API 验证有效性，显示登录用户名
- **删除 Cookie**：删除已保存的 Cookie 文件

> 采集触发前会自动预检 Cookie，未配置或已过期时采集按钮禁用。

#### UP主多选

下拉多选框，指定本次采集的 UP主范围：
- 从 DuckDB 加载已配置 UP主列表（显示名称 + 视频数）
- 支持多选、取消、清空
- 不选则采集全部 UP主

#### 触发参数

| 参数 | 说明 |
|------|------|
| 最大视频数 | 可选，限制每个 UP主采集的视频数量 |
| UP主选择 | 可选，多选指定采集对象 |

点击"开始采集"后通过 Docker SDK 启动 bilibili-monitor 容器，运行中时按钮禁用（互斥控制）。

#### 实时日志

终端风格的日志滚动窗口（深色背景、绿色字体）：
- 运行中每 5 秒自动刷新
- 完成后显示最终日志 + 退出码
- 自动滚动到最新行

#### UP主管理

嵌套在采集触发底部，用于增删 UP主配置：

**当前列表**：卡片式展示已配置 UP主
- 头像 + 名称 + 视频数 + whisper_model
- 右侧删除按钮（带确认对话框）

**添加新 UP主**：
1. 粘贴 B站主页链接（`space.bilibili.com/<uid>`）或视频链接（`bilibili.com/video/<bvid>`）
2. 选择 Whisper 模型（small / medium）
3. 点击"解析" → 展示预览卡片（头像、名称、UID）
4. 点击"确认添加" → 生成 `config/<uid>.yaml`，下次采集自动包含

### 3.2 GPU转录

分为两个区域：开发机 GPU 转录和 NAS 云 ASR 转写。

#### GPU 转录（开发机）

> 仅开发机可用，需要 NVIDIA GPU（RTX 4060）和 gpu-service 容器。

**GPU 状态卡片**：
- ✅ CUDA 可用：GPU 名称 + 显存大小 + PyTorch 版本
- ⚠️ GPU 不可用：黄色警告
- 服务不可达时显示引导信息（启动命令）

**配置区域**：
- downloads 目录路径（默认 `/sync/downloads`）
- transcripts 目录路径（默认 `/sync/transcripts`）
- 模型选择：tiny / base / small（推荐）/ medium
- 设备选择：CUDA (GPU) / CPU

**触发与进度**：
- 开始按钮（GPU 不可用时禁用）
- 进度条（成功数 / 总数）
- 实时日志（终端风格，自动滚动）
- 刷新状态按钮

#### 云 ASR 转写（NAS）

使用硅基流动 SenseVoiceSmall 模型（免费）进行云端语音转写，替代本地 Whisper。

**开关控制**：
- Toggle 开关（开启/关闭）
- 模型显示：FunAudioLLM/SenseVoiceSmall（免费）
- 开启后，bilibili-monitor 定时任务自动使用 ASR 转写

**月度预算**：
- 输入框设置分钟数上限 + 保存按钮
- 用量进度条：本月已用 / 预算（百分比）
- 预算耗尽时自动停止转写

**手动触发**：
- "手动触发 ASR 转写"按钮（ASR 未开启时禁用）
- 通过 Docker SDK 启动 `bilibili-monitor --asr`

**转写记录**：
- 最近 10 条转写记录（日期、UP主、视频标题、时长、状态）
- 按月统计总时长

### 3.3 查询日志

展示所有通过 `/api/chat` 发起的查询记录，支持分页和过滤。

#### 统计卡片（顶部）

| 卡片 | 内容 |
|------|------|
| **总查询数** | 历史累计查询次数 |
| **平均响应时间** | 所有查询的平均耗时（秒） |
| **结构化 / 语义 / 混合** | 各路由类型的查询次数和占比 |

#### 查询记录表格

| 列名 | 说明 |
|------|------|
| **问题** | 用户提问（截断显示，点击行可展开完整内容） |
| **路由类型** | 彩色标签（蓝=结构化 / 绿=语义 / 紫=混合） |
| **响应时间** | 本次查询耗时（秒） |
| **查询时间** | 发起查询的时间戳 |

#### 交互功能
- **过滤**：顶部按钮组按路由类型过滤（全部 / 结构化 / 语义 / 混合）
- **分页**：底部翻页控件，每页 15 条
- **展开**：点击行展开查看完整问题文本

### 3.4 服务监控

实时展示各 Docker 容器的运行状态和资源使用情况。

#### 容器状态卡片

每个服务一张卡片，展示：

| 指标 | 说明 |
|------|------|
| **状态灯** | 绿色圆点 = running / 红色 = stopped / 灰色 = 未知 |
| **容器名称** | Docker 容器名 |
| **内存使用** | 进度条 + 数值（如 `512 MB / 1 GB`） |
| **CPU 使用率** | 百分比显示 |
| **端口映射** | 暴露的端口列表 |

监控的服务：
1. **chromadb**（端口 8001）— 向量数据库
2. **text-to-sql**（端口 8010）— 结构化查询引擎
3. **rag**（端口 8090）— 语义检索引擎
4. **router-agent**（端口 8000）— 路由 Agent
5. **frontend**（端口 80）— 前端 + Nginx
6. **gpu-service**（端口 8011）— GPU 转录服务（仅开发机）

#### 知识库指标

| 指标 | 数据来源 |
|------|---------|
| **向量文档块数** | ChromaDB `video_knowledge` collection |
| **视频总数** | DuckDB `video_meta` 表 |
| **总查询数** | DuckDB `query_log` 表 |

#### 查询类型分布

纯 CSS 实现的水平条形图，展示结构化 / 语义 / 混合三种路由的查询占比，带颜色编码和百分比标注。

#### 刷新控制
- **自动刷新**：默认 30 秒自动刷新，可开关
- **手动刷新**：点击刷新按钮立即更新数据

## 四、API 接口

前端调用的所有后端接口（通过 Nginx 反向代理转发）：

### 核心接口（Router Agent :8000）

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `POST` | `/api/chat` | 统一问答入口 | `{ question, force_route? }` |
| `GET` | `/api/status` | 系统状态 | — |
| `GET` | `/api/up_list` | UP 主列表（DuckDB） | — |
| `GET` | `/api/recent` | 最近采集视频 | — |
| `GET` | `/api/categories` | 分类统计 | — |

### 采集触发接口

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `POST` | `/api/trigger_monitor` | 触发采集任务 | `{ max_videos?, up_names? }` |
| `GET` | `/api/trigger_status` | 采集任务状态 + 日志 | — |

### Cookie 管理接口

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `GET` | `/api/cookie` | Cookie 状态（是否配置 + 来源） | — |
| `POST` | `/api/cookie` | 保存 Cookie | `{ content }` (Netscape 格式) |
| `DELETE` | `/api/cookie` | 删除已保存的 Cookie | — |
| `POST` | `/api/cookie/test` | 测试 Cookie 有效性（B站 API） | — |

### UP主管理接口

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `GET` | `/api/up_info` | 列出所有已配置 UP主 | — |
| `POST` | `/api/up_info` | 添加新 UP主 | `{ url, whisper_model? }` |
| `DELETE` | `/api/up_info/{uid}` | 删除 UP主配置 | — |
| `GET` | `/api/up_info/resolve` | 解析链接预览 UP主信息 | `?url=xxx` |

### ASR 转写接口

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `GET` | `/api/asr/status` | ASR 设置 + 用量 + 预算 | — |
| `POST` | `/api/asr/settings` | 更新 ASR 设置 | `{ enabled?, monthly_budget_minutes? }` |
| `POST` | `/api/asr/transcribe` | 手动触发 ASR 转写 | — |

### 查询日志 / 系统监控接口

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `GET` | `/api/query_stats` | 查询统计 + 分页记录 | `?page=1&page_size=15&route_type=` |
| `GET` | `/api/system_metrics` | 系统指标 | — |

### GPU 转录接口（直连 gpu-service :8011）

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|------------|
| `GET` | `/api/gpu/status` | GPU 信息 + 任务状态 | — |
| `POST` | `/api/gpu/transcribe` | 触发 GPU 转录 | `{ downloads, transcripts, model_size, device }` |

> GPU 接口直连 `localhost:8011`，不走 Nginx 代理，仅开发机可达。

## 七、开发信息

| 项目 | 说明 |
|------|------|
| 源码位置 | `text-to-sql/frontend/` |
| 开发服务器 | `npm run dev`（端口 3000，热更新） |
| 生产构建 | `npm run build`（输出到 `dist/`） |
| API 代理 | Vite dev server 代理 `/api` → `http://localhost:8000` |
| GPU API | 直连 `http://localhost:8011`（不走 Nginx） |
| 生产部署 | Nginx 容器（端口 80），反向代理前端静态文件 + API |
| 配置文件 | `vite.config.ts`（构建）、`nginx.conf`（部署） |
